
# PRD (MVP) — AI 도구 추천/시각화 서베이 웹사이트

## 1) 제품 개요

### 제품명(가칭)

* **툴매치(ToolMatch)** / **AI툴 나침반** / **VibeTool Finder**

### 한 줄 정의

초보자가 몇 가지 질문에 답하면, **내 상황에 맞는 AI 도구 조합을 추천**하고
그 결과를 **“키워드 클러스터 + 포지셔닝 맵 + 페르소나 매트릭스”**로 **한 장 리포트**로 보여주는 서비스.

### 문제(Problem)

* 초보자는 “도구가 너무 많고” **무엇을 골라야 하는지 판단 기준이 없다**
* 숫자 점수는 신뢰가 떨어짐(근거 불명) → 대신 **반복되는 장단점/상황 적합성**이 더 설득력 있음
* 결과를 공유하거나 내부 설득(팀/상사/동료)하는 형태로 정리하기 어렵다

### 해결(Solution)

* 2~4분짜리 짧은 서베이로 사용자 상태를 분류
* 점수 대신 **상황 기반 추천 + 근거(키워드 클러스터) + 시각화(2×2 맵/매트릭스)** 제공
* 링크/이미지/PDF로 공유 가능

---

## 2) 타깃 사용자 & 핵심 유즈케이스

### Primary 타깃

1. **비개발자(창업자/PM/전략/기획)**: “데모/MVP를 빨리 만들고 싶다”
2. **주니어 개발자/솔로 개발자**: “내 레포에서 생산성을 올리고 싶다”
3. **테크리드/팀**: “에이전트 병렬처리로 개발 운영을 하고 싶다”

### 핵심 유즈케이스(Top 3)

* (U1) “내 상황에 맞는 도구 조합이 뭔지” 3분 안에 추천받기
* (U2) 결과를 **한 장 리포트**로 저장/공유하기(팀 설득/회고)
* (U3) “왜 이 도구냐”를 **키워드/근거**로 설명하기(초보자 교육)

---

## 3) MVP 범위(Scope)

### ✅ MVP에 반드시 포함

* **서베이(8~12문항)**
* 결과 페이지:

  * **추천 조합(Primary / Also)**
  * **2×2 포지셔닝 맵(정량 점수 아님)**
  * **페르소나 → 툴 매트릭스(P=주도, A=보조)**
  * **툴별 키워드 클러스터(자주 칭찬/불만)**
  * “초보자 피칭용 20초 멘트(자동 생성)”
* 결과 공유:

  * **공유 링크 생성**
  * **PNG/PDF 다운로드(최소 1개)**
* 한국어 UI (기본), 영어는 후순위 옵션

### ❌ MVP에서 제외(Non-goals)

* 실시간 가격 비교/최저가 추적
* 자동으로 커뮤니티 데이터를 크롤링해서 갱신(초기에는 수동 큐레이션)
* 로그인 필수(초기에는 익명 결과 링크 중심)

---

## 4) 서베이 설계(초안)

> 원칙: “정밀 진단”이 아니라 “초보자에게 납득되는 분기”가 목표

### 질문 카테고리

1. **역할/관점**

* 당신의 주 역할은? (PM/기획/전략, 개발, 리드/매니저, 창업자, 기타)

2. **목표**

* 가장 급한 목표는?

  * A) 아이디어 → 데모/MVP
  * B) 기존 코드에서 개발 속도↑
  * C) 리서치/기획/문서 생산성↑
  * D) 병렬 에이전트로 작업 분산/운영

3. **코드/개발 편안함**

* 코드 수정이 가능한가? (전혀/간단히/익숙/전문)

4. **팀 구성**

* 혼자/2~3명/팀(4+)

5. **산출물 형태**

* 원하는 결과는? (데모 링크/깃허브 레포/PRD·리포트/자동화된 PR)

6. **제약**

* 보안/컴플라이언스 민감도 (낮음/중간/높음)
* 선호 생태계 (Google 중심 / OpenAI 중심 / 상관없음)

7. **시간**

* 결과가 필요할 때 (오늘/이번 주/이번 달)

### 서베이 결과 분류(예시)

* Persona: Idea→Demo / PM-Research / Solo Dev / Team Lead
* Skill: Code Comfort (Low/Med/High)
* Constraint: Security (Low/High)
* Output Type: Demo vs Repo vs Doc vs Automation

---

## 5) 추천 로직(초기 버전)

### 로직 원칙

* **점수화 X** (근거 논쟁 방지)
* **룰 기반 매칭 + 설명** (왜 추천인지 문장으로 제공)
* 결과는 “추천 1개”가 아니라 **Primary 1~2개 + Also 1~2개** 조합

### 예시 규칙(초안)

* Persona=Idea→Demo & CodeComfort=Low → **Primary: Lovable**, Also: ChatGPT/Claude
* Persona=Solo Dev & CodeComfort=High → **Primary: Cursor 또는 Claude Code**, Also: ChatGPT/Claude
* Persona=Team Lead → **Primary: Codex 또는 Antigravity**, Also: Cursor/Claude Code
* Persona=PM-Research → **Primary: ChatGPT/Claude/Gemini**, Also: Lovable(프로토타입)

---

## 6) 결과 페이지 구성(정보 설계)

### (1) 상단 요약 카드

* “당신은 **[Persona]** 유형”
* “가장 잘 맞는 조합”
* “추천 이유 3줄(자동 생성)”

### (2) 시각화 1: Persona → Tool Matrix

* P/A 표로 즉시 납득

### (3) 시각화 2: 2×2 Positioning Map

* 축 예시:

  * X: No-code/Low-code → Codebase-first
  * Y: Research/Planning → Engineering/Automation
* “이건 점수 아님(qualitative positioning)” 라벨 고정

### (4) 툴 카드: 키워드 클러스터

* “자주 칭찬” / “자주 불만”
* “이 툴이 잘 맞는 상황” 1줄

### (5) 피칭용 스크립트 생성

* “20초 버전 / 60초 버전”
* 복사 버튼

### (6) 공유/내보내기

* 링크 공유
* PNG/PDF 저장

---

## 7) 콘텐츠 운영(가장 중요)

### 데이터 구조(초기)

* Tool DB

  * name, category(builder/IDE/agent/chat)
  * best_for(문장)
  * praised_keywords[], criticized_keywords[]
  * positioning(x,y) (정성 좌표)
  * persona_fit (P/A)
  * references[] (공식문서/커뮤니티 링크)

### 업데이트 방식(MVP)

* 초기에는 **수동 큐레이션 + 버전 관리**
* “최근 업데이트 날짜” 표시(신뢰 확보)

---

## 8) 성공 지표(Metrics)

### MVP 핵심 지표

* 서베이 완료율(Completion rate)
* 결과 페이지 공유율(Share rate)
* “도움 됐다” 클릭(thumbs up) 비율
* 재방문율(7일)

### 이후 확장 지표

* 팀 단위 공유(조직 내 링크 클릭 수)
* 유료 전환(리포트 템플릿/팀 대시보드)

---

## 9) 기술 요구사항(간단)

### MVP 기술 요구

* 반응형(모바일/데스크탑)
* 결과 링크 저장(서버/DB)
* PDF/PNG export
* i18n(최소 ko, 구조는 en 확장 가능)
* 접근성 기본(명확한 폰트/대비)

### 추천 스택(빠르게)

* Front: **Next.js or Vite+React** (너 기존이 Vite면 유지 추천)
* Storage: Supabase / Firebase / 간단히는 서버리스 KV
* Export: client-side canvas/PDF
* Font: **Noto Sans KR** 웹폰트로 한글 깨짐 방지

---

## 10) 리스크 & 대응

* **근거 논쟁**: “점수” 금지, “키워드 기반” + “업데이트 날짜/출처 링크”로 방어
* **신뢰 문제**: “정답이 아니라 가이드” 고정 문구 + 사용자가 결과를 조정 가능(옵션)
* **저작권**: 리뷰 원문 인용 최소화(짧은 요약/키워드 중심), 링크로 연결
* **최신성**: “최근 업데이트”와 “버전” 노출

---

## 11) MVP 일정(현실적인 최소)

* Day 1: 정보구조/서베이 확정 + 결과 템플릿 확정
* Day 2~3: 프론트(서베이/결과 UI) + 룰 엔진
* Day 4: 공유 링크/저장 + export(PNG/PDF)
* Day 5: 콘텐츠 튜닝(키워드/문구) + QA + 배포
